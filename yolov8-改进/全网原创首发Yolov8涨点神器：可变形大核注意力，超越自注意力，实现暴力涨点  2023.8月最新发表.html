1.可变形大核注意力(D-LKA Net)

论文：arxiv.org/pdf/2309.00121.pdf

 摘要：医学图像分割通过 Transformer 模型得到了显着改进，该模型在掌握深远的上下文和全局上下文信息方面表现出色。 然而，这些模型不断增长的计算需求（与平方令牌数量成正比）限制了它们的深度和分辨率能力。 目前的大多数方法都是逐片处理 D 体图像数据（称为伪 3D），缺少关键的片间信息，从而降低了模型的整体性能。 为了应对这些挑战，我们引入了可变形大核注意力（D-LKA Attention）的概念，这是一种采用大卷积核来充分理解体积上下文的简化注意力机制。 这种机制在类似于自注意力的感受野中运行，同时避免了计算开销。 此外，我们提出的注意力机制受益于可变形卷积来灵活地扭曲采样网格，使模型能够适当地适应不同的数据模式。我们设计了 D-LKA 注意力的 2D 和 3D 适应，后者在跨深度方面表现出色 数据理解。 这些组件共同塑造了我们新颖的分层 Vision Transformer 架构，即 D-LKA Net。根据流行的医学分割数据集（Synapse、NIH 胰腺和皮肤病变）上的领先方法对我们的模型进行的评估证明了其卓越的性能。

为了解决这些挑战，我们引入了Deformable Large Kernel Attention (D-LKA Attention)}的概念，这是一种采用大卷积核来充分理解体积上下文的简化注意力机制。

这种机制在类似于自注意力的感受野中运行，同时避免了计算开销。 此外，我们提出的注意力机制受益于可变形卷积来灵活地扭曲采样网格，使模型能够适当地适应不同的数据模式。 我们设计了 D-LKA Attention 的 2D 和 3D 适应，后者在跨深度数据理解方面表现出色。

  161. def autopad(k, p=None, d=1):  # kernel, padding, dilation
  162.     # Pad to 'same' shape outputs
  163.     if d > 1:
  164.         k = d * (k - 1) + 1 if isinstance(k, int) else [d * (x - 1) + 1 for x in k]  # actual kernel-size
  165.     if p is None:
  166.         p = k // 2 if isinstance(k, int) else [x // 2 for x in k]  # auto-pad
  167.     return p

  170. class Conv(nn.Module):
  171.     # Standard convolution with args(ch_in, ch_out, kernel, stride, padding, groups, dilation, activation)
  172.     default_act = nn.SiLU()  # default activation
  173.  
  174.     def __init__(self, c1, c2, k=1, s=1, p=None, g=1, d=1, act=True):
  175.         super().__init__()
  176.         self.conv = nn.Conv2d(c1, c2, k, s, autopad(k, p, d), groups=g, dilation=d, bias=False)
  177.         self.bn = nn.BatchNorm2d(c2)
  178.         self.act = self.default_act if act is True else act if isinstance(act, nn.Module) else nn.Identity()
  179.  
  180.     def forward(self, x):
  181.         return self.act(self.bn(self.conv(x)))
  182.  
  183.     def forward_fuse(self, x):
  184.         return self.act(self.conv(x))

  186. class C2f(nn.Module):
  187.     # CSP Bottleneck with 2 convolutions
  188.     def __init__(self, c1, c2, n=1, shortcut=False, g=1, e=0.5):  # ch_in, ch_out, number, shortcut, groups, expansion
  189.         super().__init__()
  190.         self.c = int(c2 * e)  # hidden channels
  191.         self.cv1 = Conv(c1, 2 * self.c, 1, 1)
  192.         self.cv2 = Conv((2 + n) * self.c, c2, 1)  # optional act=FReLU(c2)
  193.         self.m = nn.ModuleList(Bottleneck(self.c, self.c, shortcut, g, k=((3, 3), (3, 3)), e=1.0) for _ in range(n))
  194.  
  195.     def forward(self, x):
  196.         y = list(self.cv1(x).chunk(2, 1))
  197.         y.extend(m(y[-1]) for m in self.m)
  198.         return self.cv2(torch.cat(y, 1))
  199.  
  200.     def forward_split(self, x):
  201.         y = list(self.cv1(x).split((self.c, self.c), 1))
  202.         y.extend(m(y[-1]) for m in self.m)
  203.         return self.cv2(torch.cat(y, 1))
  204.  
  205. class Bottleneck(nn.Module):
  206.     # Standard bottleneck
  207.     def __init__(self, c1, c2, shortcut=True, g=1, k=(3, 3), e=0.5):  # ch_in, ch_out, shortcut, groups, kernels, expand
  208.         super().__init__()
  209.         c_ = int(c2 * e)  # hidden channels
  210.         self.cv1 = Conv(c1, c_, k[0], 1)
  211.         self.cv2 = Conv(c_, c2, k[1], 1, g=g)
  212.         self.add = shortcut and c1 == c2
  213.  
  214.     def forward(self, x):
  215.         return x + self.cv2(self.cv1(x)) if self.add else self.cv2(self.cv1(x))
  216.

  224.         super().__init__()

  226.         self.cv1 = Conv(c1, 2 * self.c, 1, 1)

  237.         y.extend(m(y[-1]) for m in self.m)
  238.         return self.cv2(torch.cat(y, 1))
